your_health.server_code.DatabaseProxy module¶

Server-side routines related to accessing data tables.

Notes

  1. Naming conventions for data table names & rows:

    1. The data tables that store user data start with user_x where x is the characteristic stored in the row. For example: user_allergies. 2. The data table itself should always be referred to in the plural.

    Do use: user_allergies Don’t use: user_allergy

    1. user_x_row is a single row in the user_x data table.

      Do use: user_allergies_row Don’t use: user_allergy_row (since user_allergy is not a data table)

    2. A shortcut to a row in user_x uses the singular form of x and is not prefixed with user_

      Do use: allergy (which is short for user_allergies_row) don’t use: allergy_row (since allergy by itself already means it is a row)

  2. Preferred order of related routines is CRUD based

    add (create), get (read), update, delete, validate

your_health.server_code.DatabaseProxy.get_all_user_items()¶

Return all data for current user for display in a summary form.

Returns

data_dict – Augments and returns table_dict for entries where in_summary=True by appending a new key ‘data’ which is a list of dictionaries suitable for a repeating panel form component.

For example, the user allergies key/values would be:

{‘user_allergies’:
{‘columns’: [“name”, “description”, “notes”],
’data’: [ {‘name’: value,

’description’: value, ‘notes’: value

} … for each data row]

… for each table in table_dict }

Return type

dict

your_health.server_code.DatabaseProxy.get_user_items(data_table_name, column_names)¶

Return rows and column data associated with columns_names from data_table_name for the current user as a list of dicts.

Parameters

  • data_table_name (str) – Data table name to extract data from. For example, ‘user_allergies’

  • column_names (list(str)) – Columns to return from data_table_name For example, [‘name’, ‘description’, ‘notes’]

Returns

data_dict – Row and associated columns from data_table_name for the current user. For example, data_dict could be a list of dicts with format:

[ {‘table_row’: row,

’name’: row[‘name’], ‘description’: row[‘description’], ‘notes’: row[‘notes’]} … ]

Return type

list(dict)
