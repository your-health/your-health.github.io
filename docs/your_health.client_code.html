class your_health.client_code.Logger.Logger(enabled=True, name='', timestamp=False)¶

Bases: object

Logger to assist in debugging if your code does not have access to the python logging module (https://docs.python.org/3.8/library/logging.html).

  1. Create a logger, determine if the logger name and/or a timestamp should be included with each log. Example: log_client_call = Logger(enabled=True, name=’client call’, timestamp=True)

Notes

  1. Based on https://anvil.works/forum/t/how-do-people-tend-to-do-debugging-client-side/4061

  2. datetime.now() the form: “2021-10-22 09:16:45.080579” Logged timestamps will strip the date and the last two digits of precision. For the datetime example above, the following time would be logged: “09:16:45.0805”

class your_health.client_code.Timer.Timer(timer_name='timer', **properties)¶

Bases: object

Timer class to aid in timing/profiling client & server code.

Will provide statistics about a timed task including:

  1. Total run-time,

  2. number of laps,

  3. min lap time,

  4. max lap time, and

  5. average lap time

  1. Create an instance of a timer (which will automatically start the timer).

    For example: my_timer = MyTimer(timer_name)

  2. Optionally restart the timer.

    For example: my_timer.start()

  3. Save a ‘lap’ time to indicate some unit of progress with the option to print diagnostics or annotate the lap with a note. For example: my_timer.lap(print_results=True, note=’working on big task’)

  4. Stop the timer to evaluate timing statistics. For example: stats = my_timer.stop(note=’completed big task’)

Created by: Tony H Created on: 2021-10-25

class your_health.client_code.Utilities.DatabaseFormTemplate¶

Bases: object

Class with uniform form, component, database routines to allow user to view/update a data table.

In the form that inherits this class you need to do the following:

  1. Set self.table_info dictionary for your table, columns, naming. For example:

    self.table_info = {‘name’: “user_allergies”,

    ‘singular’: “allergy”, ‘plural’: “allergies”, ‘columns’: [“name”, “description”, “notes”]}

  2. Setting the columns key results in the creation of 3 self.item keys for each item in the columns list to associate with form components.

    For example, for the self.column value ‘phone’, the follow keys are created:

    self.item[‘phone’]: phone value saved or cached from the database self.item[‘phone new’]: proposed phone value for a new database row self.item[‘phone edit’]: proposed phone edited value for an existing database row

  3. Create the following column container(s) in your Form:

    1. self.new_column_panel

  1. Put labels, textboxes associated with new data rows in self.new_column_panel

  2. Set self.new_column_panel textboxes bindings to the column of interest + ” new”

    For example, if you want the textbox to be associated with “first name” then bind it to “first name new”

  3. Make sure to enable write-backs for the bound textboxes

  4. Set the component you wish to have focus in the form_show method

    e.g., self.new_note_textbox.focus()

Notes

  1. pagination gets weird when you add/delete, i have disabled paging and navigation arrows.

class your_health.client_code.Utilities.DatabaseRowTemplate¶

Bases: object

Class with uniform form, component, database routines to allow user to view/update a single row of repeating_panel.items.

See DatabaseFormTemplate for additional information.

In the form that inherits this class you need to do the following:
  1. Create the following column container(s) in your Form

    1. self.view_column_panel

    2. self.edit_column_panel

  2. Put labels associated with view data rows in self.view_column_panel

    1. Set bindings for labels in this container to the the column of interest from the list of self.table_info[‘columns’]. For example, if self.table_info[‘columns’] has ‘phone’ as a member set the data binding for the labels to self.item[‘phone’].

  3. Put textboxes associated with edit data rows in self.edit_column_panel

    1. Set bindings for textboxes in this container to the the column of interest from the list of self.table_info[‘columns’] + ‘ edit’. For example, if self.table_info[‘columns’] has ‘phone’ as a member set the data binding for the textboxes to self.item[‘phone edit’].

Notes

  1. To avoid having edited but unsaved data lost on sorting/refreshing of other rows, item uses the unsaved_edits and other keys to retain state.

  2. self.item[‘unsaved_edits’] is a flag to indicate unsaved data are present.
